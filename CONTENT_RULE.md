## ⚠️ ファイル優先順位

このファイル（CONTENT_RULE.md）は、**PROJECT_RULE.mdを最優先とした上で**、
コンテンツ制作工程（テーマ選定〜セルフレビュー）を担当するAIの行動ルールを定める。

PROJECT_RULE.mdと本ファイルの内容が矛盾する場合は、**PROJECT_RULE.mdを優先する。**

本ファイルが担当する工程：

- テーマ選定
- タイトル
- キャプション
- ハッシュタグ
- 画像設計書
- リール構成
- セルフレビュー

本ファイルが担当しない工程：

- 画像生成（一切関与しない）

## 1. 作業開始前の確認事項

作業を始める前に必ず以下を確認する。

```
□ PROJECT_RULE.mdを最新版で取得したか
□ 今月の月間テーマは何か（PROJECT_RULE.md セクション16）
□ 今回の投稿種類はどれか（7-A〜7-E）
□ 前回の投稿テーマと重複していないか
```

確認後、以下の「作成前一覧」フォーマットで出力してから本文制作に入る。

## 3. テーマ選定ルール

### 選定基準（優先順）

1. 今月の月間テーマに関連しているか
2. 株初心者がつまずきやすい場面か
3. 前回の投稿と概念が重複していないか
4. 一投稿で扱う概念が1つに絞られているか

### テーマ選定の出力形式

テーマを選定した場合、以下の形式で理由も併記する。

```
テーマ：〇〇
選定理由：〇〇（月間テーマとの関連、初心者のつまずきポイントなど）
投稿種類：7-〇（〇〇形式）
```

## 4. タイトル制作ルール

### 基本ルール（PROJECT_RULE.md セクション8より）

- 20文字以内を厳守
- 疑問形・あるある系を優先
- 感情・状況を切り取った表現にする
- 説明的なタイトルは禁止

### タイトル制作の手順

1. テーマを一言で表す「状況・感情ワード」を決める
2. 「〜、〇〇？」または「〜って〇〇？」の形に整える
3. 文字数を数える（20文字以内か確認）
4. 初心者が「自分のことだ」と思えるか確認する

### タイトル案の出し方

タイトルは必ず**3案以上**出してから最良案を選ぶ。

```
タイトル案①：〇〇（〇文字）
タイトル案②：〇〇（〇文字）
タイトル案③：〇〇（〇文字）
採用：タイトル案〇「〇〇」
採用理由：〇〇
```

## 5. キャプション制作ルール

### 共通ルール（PROJECT_RULE.md セクション9より）

- 冒頭は必ず共感フックから入る（CTAから始めない）
- 結論はフックの直後に出す
- 専門用語は初出時に1行以内で説明する
- 共感を1回以上入れる
- 1文ごとに改行する
- CTAは末尾に1回のみ（冒頭・中間への配置禁止）
- 最後は前向きに終える

### 投稿種類別の文字数と構成

文字数は作成後に必ずカウントして記載する。

#### 7-A 問題形式投稿（300〜400文字）

```
1. 共感フック（初心者が感じる場面描写）
2. 結論・答え
3. 専門用語の説明（初出のもののみ・1行以内）
4. ポイント整理（3つ以内）
5. 前向きな締め
6. CTA（末尾のみ・固定文言）
```

#### 7-B 初心者あるある（300〜400文字）

```
1. 共感フック（「〜ですよね😂」など感情に寄り添う）
2. でも〇〇できたことは成長（具体的に何ができたかを書く）
3. 次に見るべきポイント（1〜2文・簡潔に）
4. 前向きな締め
5. CTA（末尾のみ・固定文言）
```

> ⚠️ ポイントは1〜2文に収める。説明が膨らんだら削る。

#### 7-C 出来高・考え方解説（350〜450文字）

```
1. 共感フック（「〜って難しいですよね」）
2. 結論（一言で）
3. 用語説明（初出の専門用語を1行で）
4. 見るべきポイント（3つ以内）
5. 前向きな締め
6. CTA（末尾のみ・固定文言）
```

#### 7-D アプリ体験投稿（200〜300文字）

```
1. 共感フック（「練習したいけど何から始めれば…」など）
2. アプリで何ができるかを1〜2文で
3. CTA（末尾のみ・強め・固定文言）
```

#### 7-E リール（150文字以内・厳守）

```
1. 共感1文
2. 結論1文（「答えは動画で確認しましょう」など）
3. CTA（末尾のみ・固定文言）
```

> ⚠️ リールキャプションに専門用語の説明を入れない。動画・ナレーションに任せる。

### CTA固定文言

```
少しずつ経験して、一緒にレベルアップしていきましょう🔥

無料株特訓アプリ
→ @chart.training
```

## 6. ハッシュタグ制作ルール

### 基本ルール（PROJECT_RULE.md セクション10より）

- 5〜10個（厳守）
- 株初心者・投資初心者向け中心
- 改行なし・横並びで記載

### 選定の考え方

```
・必須タグ（毎回入れる）：#株初心者 #投資初心者 #株式投資
・テーマタグ（今回の内容に合わせて2〜4個）：例 #ローソク足 #出来高 #チャート分析
・あるある・感情タグ（7-B投稿に合わせて）：例 #株あるある #投資あるある #メンタル管理
・初心者属性タグ（1〜2個）：例 #投資勉強中 #初心者投資家
```

合計が10個を超えないようにテーマタグと属性タグで調整する。

## 7. 画像設計書ルール

### 担当範囲の確認

本ファイルが担当するのは**設計書の記述のみ**。

画像生成・加工・出力は担当しない。

設計書は画像生成担当（Codexなど）への指示書として機能する。

### 設計書の記述ルール

- 画像1枚ごとに1つの設計書を書く
- 構成要素・テキスト内容・レイアウトの意図を明記する
- 「何を見せるか」を書く（「どう生成するか」は書かない）
- 画像内に入れてはいけない要素を明記する

### 設計書フォーマット

```
【画像設計書①】
用途：（問題 / 答え / 解説+CTA / 6コマ漫画 など）
サイズ：（例：1080×1350px）
見せたいこと：（この画像で初心者に伝えたい1点）
テキスト要素：（画像内に入れるテキストを列記・3〜5点まで）
入れてはいけないもの：（キャプション本文 / ハッシュタグ / 答えの文言 など）
補足：（レイアウト・強調ポイントの意図など）
```

### 投稿種類別の画像枚数（PROJECT_RULE.md セクション7より）

| 投稿種類 | 画像枚数 | 内容 |
|---|---|---|
| 7-A 問題形式 | 3枚 | ①問題 ②答え ③解説+CTA |
| 7-B 初心者あるある | **1枚** | ①6コマ漫画（完結型） |
| 7-C 解説 | 3枚 | ①概念図 ②ポイント整理 ③まとめ+CTA |
| 7-D アプリ体験 | 3枚 | ①アプリ画面 ②使い方の流れ ③CTA強調 |
| 7-E リール | 3枚 | ①問題（0〜3秒） ②答え（10〜18秒） ③解説+CTA（18〜30秒） |

> ⚠️ 画像枚数を増やす提案は禁止。

### 画像①（問題形式・リール）の禁止事項

- 答えの文言を入れない
- 「正解は〇〇」「答え：〇〇」に見える表現を入れない

### 画像②（問題形式・リール）のルール

- アプリ画面・CTA要素は入れない
- 答えと確認ポイント（チェックリスト）のみに絞る

### 画像③のルール

- CTA・アプリ導線はこの画像にのみ入れる
- 画像②に入っていた場合は設計書の段階で修正指示を出す

### 6コマ漫画の構成（7-B専用）

```
コマ①：状況発生（問題・失敗の場面）
コマ②：初心者の行動（あるある）
コマ③：予想外の結果（どんでん返し・読者が「えっ！？」となる展開）
コマ④：落ち込み・後悔
コマ⑤：でも〇〇はできた（前向きな気づき）
コマ⑥：次はこうしよう（改善・行動）
```

> ⚠️ コマ③は「どんでん返し」として機能させる。
> コマ②の繰り返しや感情の強調だけでは不十分。
> 読者が「あるある！でもそうなるの！？」と感じる展開を必ず入れる。

## 8. リール構成ルール

### 担当範囲

- ナレーション構成（秒数・内容・台本）
- 画像設計書（上記セクション7に従う）

### ナレーション構成フォーマット

```
0〜3秒：問題提示（「〜、どうする？」）
3〜10秒：共感・間（「〜ですよね。〜と思いやすい場面です。」の2文程度）
10〜18秒：答え発表（「答えは、〇〇。」）
18〜24秒：解説（3点以内・簡潔に）
24〜30秒：CTA（「練習したい人はプロフィールの無料アプリへ。」）
```

### リール制作の注意事項

- キャプションに解説を書かない（動画・ナレーションに任せる）
- ナレーション3〜10秒の共感パートは2文以上入れる
- 専門用語はナレーション内で説明する（初出時に1文で）
- CTA画像（③）と最後のナレーションは内容を一致させる

## 9. セルフレビュー

### 実施タイミング

投稿本文・設計書・リール構成を出力した直後に必ず実施する。

### セルフレビューの形式

以下の2段階で実施する。

#### ステップ1：チェックリスト

```
【共通チェック】
□ PROJECT_RULE.mdに矛盾する箇所がないか
□ 初心者向けか
□ 専門用語の説明が初出時に1行以内であるか
□ 断定的な投資表現がないか
□ 前向きな内容か
□ 説教していないか

【形式チェック】
□ タイトルが20文字以内か（文字数を記載する）
□ キャプションが種類別の文字数範囲内か（文字数を記載する）
□ ハッシュタグが5〜10個か（個数を記載する）
□ CTAが末尾1回のみか
□ 画像枚数が投稿種類の規定通りか

【保存率チェック】
□ タイトルだけで続きを見たくなるか
□ 共感ポイントがあるか
□ 初心者の悩みに答えているか
□ 行動したくなる内容か
```

#### ステップ2：改善提案

チェックリストで「△」または「×」があった場合、改善案を**3案以上**出す。

問題がない場合も、「さらに良くなる余地」を1点以上挙げる。

```
【改善提案】
1. 〇〇について
　案A：〇〇
　案B：〇〇
　案C：〇〇
```
