# REVIEW_RULE.md
# レビューAI専用 評価ルール

最終更新: 2026-06-27

> このファイルはレビューAIが**投稿を評価する方法**だけを定義する。
> 投稿のルール・仕様・制作工程の参照先は以下の通り。
>
> | 確認内容 | 参照先 |
> |---|---|
> | 投稿の方針・禁止事項・種類別形式 | PROJECT_RULE.md |
> | 制作工程・出力フォーマット | CONTENT_RULE.md |
> | 画像仕様・デザイン基準 | IMAGE_RULE.md |

---

## 1. レビューの目的

**誰がレビューしても同じ評価になること。**

- 主観・好みで採点しない
- 「なんとなく良い・悪い」は減点根拠にしない
- **ルールに照らして違反があるか／ないか** を判定する
- 違反がなければ減点しない。違反があれば必ず根拠ルールを示す

---

## 2. 採点方法

### 基本ルール
- **100点満点・減点方式**
- 満点スタートで、違反・不足があるたびに減点する
- 良い点は採点に加算しない・コメントしない
- 減点根拠のないマイナスは禁止

### 採点カテゴリと配点

| カテゴリ | 配点 | 内容 |
|---|---|---|
| A. コンプライアンス | 25点 | 断定表現・禁止テーマ・投資助言の有無 |
| B. 投稿形式 | 25点 | タイトル・キャプション文字数・ハッシュタグ・CTA・枚数 |
| C. 初心者目線 | 25点 | 理解しやすさ・共感・専門用語説明 |
| D. 保存・行動誘発 | 15点 | 保存したくなるか・行動につながるか |
| E. 画像チェック | 10点 | 画像仕様・テキスト点数・禁止事項 |

---

## 3. 減点基準

### A. コンプライアンス（25点）

| 違反内容 | 減点 |
|---|---|
| 断定的な投資表現が1箇所ある（「必ず」「絶対」「買いサイン」等） | **－10点** |
| 断定的な投資表現が2箇所以上ある | **－25点**（上限） |
| 禁止テーマが含まれる（信用取引・FX・特定銘柄推奨等） | **－25点**（上限） |
| 「必ず」「絶対」「確実に」が投資成果に関して使われている | **－10点／箇所** |

> ⚠️ コンプライアンス違反が1件でもある場合、他カテゴリの採点前に必ず指摘する。

### B. 投稿形式（25点）

| 違反内容 | 減点 |
|---|---|
| タイトルがない | **－8点** |
| タイトルが20文字を超えている | **－4点** |
| キャプションの文字数が種類別範囲を外れている（超過・不足どちらも） | **－6点** |
| ハッシュタグが5個未満または10個超 | **－4点** |
| ハッシュタグが改行されている | **－2点** |
| CTAがない | **－6点** |
| CTAが末尾以外にある（冒頭など） | **－4点** |
| CTAが2箇所以上ある | **－4点** |
| 投稿種類に対応した画像枚数と合っていない | **－6点** |
| 7-B 初心者あるあるでイラスト案②③が存在する | **－6点** |

### C. 初心者目線（25点）

| 違反内容 | 減点 |
|---|---|
| 専門用語の初出説明がない（1語につき） | **－4点** |
| 上から目線・説教口調になっている | **－8点** |
| 共感ポイントが1箇所もない | **－6点** |
| 初心者を不安にさせる表現がある | **－6点** |
| 1投稿に概念が2つ以上含まれている（詰め込み） | **－6点** |
| キャプション冒頭がCTAまたは広告的な文言で始まっている | **－4点** |

### D. 保存・行動誘発（15点）

| 違反内容 | 減点 |
|---|---|
| タイトルを読んでも続きを見たくならない（説明的すぎ・長すぎ） | **－5点** |
| 説明だけで終わっており、共感・行動につながっていない | **－5点** |
| 前向きな締めがない | **－5点** |

### E. 画像チェック（10点）

| 違反内容 | 減点 |
|---|---|
| キャプションまたはハッシュタグが画像内に含まれている | **－5点** |
| 画像1枚あたりのテキスト点数が6点以上ある | **－3点** |
| 画像サイズが仕様と異なる（フィード：1080×1350 / リール：1080×1920） | **－3点** |
| テーマカラー以外の原色がメインで使われている | **－2点** |
| フォントが3種類以上混在している | **－2点** |
| ファイル名が命名規則に従っていない | **－2点** |

> 画像が提出されていない場合、カテゴリEは「評価対象外（－0点）」として扱う。

## 4. 必須チェック項目

レビュー開始時に以下を必ず先に確認する。**1つでも該当したらその時点で明記する。**

```
□ 投稿種類（7-A〜7-E）が特定できるか
□ コンプライアンス違反（断定表現・禁止テーマ）がないか
□ CTAの固定文言が正確に使われているか
□ 画像枚数が投稿種類と一致しているか
```

## 5. 投稿種類ごとのチェック項目

### 7-A 問題形式

| チェック項目 | 判定基準 |
|---|---|
| イラスト案①に答えの文言がないか | 問題のみ・「あなたならどうする？」形式か |
| イラスト案②に結論が大きく表示されているか | チェックリスト形式になっているか |
| イラスト案③にポイント整理とCTAがあるか | 3点以内に整理されているか |
| キャプションにポイント整理があるか | 3つ以内か |
| キャプション文字数 | 300〜400文字以内か |

### 7-B 初心者あるある

| チェック項目 | 判定基準 |
|---|---|
| イラスト枚数が1枚か | 2枚以上は即減点 |
| 6コマ構成になっているか | 6コマ全てに役割があるか |
| 6コマの流れが正しいか | 状況→行動→結果→後悔→気づき→改善の順か |
| キャプションが共感フックから始まっているか | 「〜ですよね」等の感情寄り添いがあるか |
| キャプション文字数 | 300〜400文字以内か |

### 7-C 出来高・考え方解説

| チェック項目 | 判定基準 |
|---|---|
| 概念が1つに絞られているか | 複数概念の詰め込みがないか |
| イラスト案①が概念図・チャートイメージか | 具体的なビジュアルになっているか |
| イラスト案②がポイント整理か | チェックリストまたは比較表形式か |
| 見るべきポイントが3つ以内か | 4点以上あれば減点対象 |
| キャプション文字数 | 350〜450文字以内か |

### 7-D アプリ体験投稿

| チェック項目 | 判定基準 |
|---|---|
| アプリの機能説明が1〜2文以内か | 説明が長すぎないか |
| イラスト案③にCTAが強調されているか | 「プロフィールの無料アプリへ」が目立つか |
| キャプションのCTAが強めか | 他種類より明確な誘導文になっているか |
| キャプション文字数 | 200〜300文字以内か |

### 7-E リール

| チェック項目 | 判定基準 |
|---|---|
| キャプションが150文字以内か | 超過は即減点 |
| キャプションが「共感1文＋結論1文＋CTA」の構成か | 専門用語の説明がキャプションに入っていないか |
| イラスト案①に答えの文言がないか | 0〜3秒用・問題のみか |
| ナレーション構成が時間軸に沿っているか | 0〜30秒の流れが正しいか |

## 6. 画像チェック項目

> 画像が提出された場合のみ実施する。画像なしの場合はこのセクションをスキップし「画像未提出のため評価対象外」と明記する。

| チェック項目 | 判定基準 | 参照 |
|---|---|---|
| 画像サイズ | フィード：1080×1350 / リール：1080×1920 | IMAGE_RULE.md § 1 |
| 画像枚数 | 投稿種類ごとの規定枚数と一致しているか | IMAGE_RULE.md § 2 |
| テキスト点数 | 1枚あたり3〜5点以内か | PROJECT_RULE.md § 11 |
| キャプション・ハッシュタグの混入 | 画像内に含まれていないか | PROJECT_RULE.md § 11 |
| テーマカラー | #2E7D32・#FF8F00・#FFFFFFの3色以内か | IMAGE_RULE.md § 4 |
| フォント種類 | 3種類以内か・明朝体・装飾フォント使用がないか | IMAGE_RULE.md § 5 |
| 本文テキストサイズ | 28px以上か | IMAGE_RULE.md § 5 |
| ファイル名 | 命名規則に従っているか（例：`qa_deadcross_01.png`） | IMAGE_RULE.md § 9 |
| 保存形式 | PNG・sRGB・透過なしか | IMAGE_RULE.md § 10 |
| ファイルサイズ | 8MB以内か | IMAGE_RULE.md § 10 |

## 7. 改善案の出し方

### 出力ルール

- **改善案は優先順位①②③の順で必ず3案出す**
- 「良かった点」は書かない
- 改善案には必ず「何をどう変えるか」を具体的に書く
- 「〜すると良い」ではなく「〜に変える」と言い切る形で書く
- 減点ゼロの項目への改善提案は禁止（違反がないものに手を加えない）

### 優先順位の決め方

| 優先度 | 基準 |
|---|---|
| ① | 減点幅が最も大きい違反・またはコンプライアンス違反 |
| ② | 次に減点幅が大きい違反・または投稿形式の不備 |
| ③ | 減点幅は小さいが読者体験に影響する改善点 |

> 同点の場合はコンプライアンス → 投稿形式 → 初心者目線 → 保存率 → 画像の順で優先する。

### 改善案の書き方フォーマット

```
① [優先度最高の改善点]
   現状：〇〇という表現になっている
   改善：〇〇に変える
   理由：PROJECT_RULE.md § 〇〇 の〇〇に違反しているため

② [次の改善点]
   現状：〇〇
   改善：〇〇に変える
   理由：〇〇

③ [3番目の改善点]
   現状：〇〇
   改善：〇〇に変える
   理由：〇〇
```

## 9. レビュー実施の原則

- レビューは**投稿種類を特定してから開始する**（種類不明の場合は確認してから進める）
- 1回のレビューで評価する対象は**1投稿分**（複数投稿をまとめて採点しない）
- 画像と文章が両方提出された場合は**両方をセットでレビューする**
- 「なんとなく良い・雰囲気が出ている」は採点根拠にしない
- 改善案を出した後に追加の「良い点コメント」を加えない
