# IMAGE_RULE.md
# Codex専用 画像生成ルール

最終更新: 2026-06-27

> このファイルは画像生成AIツール（Codex）が画像を作成する際の仕様を定義する。
> 投稿内容・キャプション・ハッシュタグのルールは **PROJECT_RULE.md を参照**すること。
> 共通イラスト方針（向き・枚数・テキスト点数・キャプション禁止など）は **PROJECT_RULE.md § 11** が最優先。

---

## 1. 画像サイズ

| 投稿種類 | 推奨サイズ（px） | アスペクト比 |
|---|---|---|
| フィード投稿（全種類） | 1080 × 1350 | 4:5（縦型） |
| リール サムネイル | 1080 × 1920 | 9:16（縦型） |
| リール 画面内スライド | 1080 × 1920 | 9:16（縦型） |

- 解像度：**72dpi以上**（Instagram表示基準）
- 拡大・縮小後に文字がつぶれないよう、テキストは余白を十分に確保して配置する

---

## 2. 画像枚数

| 投稿種類 | 枚数 | 備考 |
|---|---|---|
| 7-A 問題形式 | 3枚 | 問題 / 答え / 解説＋CTA |
| 7-B 初心者あるある | **1枚** | 6コマ漫画を1枚に収める |
| 7-C 出来高・考え方解説 | 3枚 | 概念図 / ポイント整理 / まとめ＋CTA |
| 7-D アプリ体験 | 3枚 | アプリ画面 / 使い方の流れ / CTA強調 |
| 7-E リール | 3枚（スライド） | 問題 / 答え / 解説＋CTA |

> 枚数を増やす提案は禁止。指定枚数で完結させること。

---

## 3. デザイン

### 基本方針
- **情報を詰め込まない**：1枚で伝えることを1〜2点に絞る
- **余白を広く取る**：コンテンツ領域は画面全体の70〜80%以内に収める
- **見出し → 本文 → 補足**の視線誘導を意識する
- ゆるい漫画風タッチ・親しみやすいイラストを優先する
- リアル写真・クールなビジネス系デザインは使わない

### 枠・背景
- 背景は白または薄いオフホワイト（#FAFAFA 前後）を基本とする
- 各画像の外周に**16〜24px 程度の余白（マージン）**を設ける
- 枠線を使う場合は細め（2〜3px）・角丸（8〜16px）にする

---

## 4. 色使い

### テーマカラー（3色以内で統一）

| 役割 | カラーコード | 用途 |
|---|---|---|
| メインカラー | #2E7D32（深緑） | 見出し・アクセント・ボタン |
| サブカラー | #FF8F00（オレンジ） | 強調・アイコン・吹き出し枠 |
| ベースカラー | #FFFFFF（白） | 背景・余白 |

### 補助カラー（使用は最小限）

| 役割 | カラーコード | 用途 |
|---|---|---|
| テキスト本文 | #212121（ほぼ黒） | 本文テキスト全般 |
| 薄グレー | #EEEEEE | 区切り線・背景の差し色 |
| 注意・マイナス | #C62828（深赤） | 禁止・NG・損切りなど強調 |

### 禁止
- テーマカラー以外の原色をメインに使わない
- グラデーションを背景全面に使わない（部分的な差し色は可）
- 1枚に4色以上のカラーを使わない

---

## 5. フォント

### 使用フォント

| 用途 | フォント | ウェイト |
|---|---|---|
| 見出し・タイトル | Noto Sans JP | Bold（700） |
| 本文・説明テキスト | Noto Sans JP | Regular（400） |
| 強調ワード | Noto Sans JP | Bold（700）＋メインカラー |
| 英数字（チャート数値など） | Noto Sans JP または Inter | Regular〜Bold |

### サイズ目安（1080px幅基準）

| 役割 | フォントサイズ |
|---|---|
| 大見出し | 52〜64px |
| 中見出し | 36〜44px |
| 本文 | 28〜34px |
| 補足・注釈 | 22〜26px |

### 禁止
- 手書き風フォント・明朝体・装飾フォントは使わない
- 1枚に3種類以上のフォントファミリーを混在させない
- 文字の縁取り（stroke）は太すぎない（最大2px程度まで）

---

## 6. キャラクター

### 基本設定
- **会社員・サラリーマン男性**をメインキャラクターとして使用する
- 年齢感：20〜40代・普通体型・ジャケットまたはカジュアル
- 表情はシンプルな2〜3頭身のゆるいデフォルメキャラ
- 感情は表情と吹き出しで表現する（驚き・困惑・納得・喜びを多用）

### 表情パターン（共通で使いまわす）

| 場面 | 表情 |
|---|---|
| 失敗・後悔 | 汗・下向き口 |
| 疑問・困惑 | ？マーク・首を傾げる |
| 納得・気づき | ！マーク・目が輝く |
| 喜び・前向き | ガッツポーズ・笑顔 |
| 考え中 | 頬に手・上向き目線 |

くまキャラはシリーズ共通キャラクター。
理由が無い限り削除しない。

### 禁止
- リアルな人物写真・有名人に似たキャラクターは使わない
- 女性キャラをメインにしない（サブとして登場は可）
- キャラクターに特定の証券会社ロゴ・銘柄名を持たせない

---

## 7. 手の描写

- キャラクターの手は**3〜4本指のシンプル描写**でよい（写実的な5本指は不要）
- 指差し・グー・パー・ガッツポーズなどポーズは明確にする
- 手が小さすぎてポーズが判別できないサイズにしない
- チャートを指差す場面では、指の向きと指先が指す対象を一致させる
- 吹き出しを持つ手はキャラクターの体と自然につながるように描く

---

## 8. レイアウト

### 基本グリッド
- 画面を**上部・中部・下部の3ゾーン**に分割して設計する

| ゾーン | 高さの目安 | 内容 |
|---|---|---|
| 上部（ヘッダー） | 全体の15〜20% | タイトル・コマ番号（漫画）・問題ラベル |
| 中部（メイン） | 全体の60〜70% | イラスト・チャート図・テキスト本文 |
| 下部（フッター） | 全体の10〜15% | アカウント名・コマ番号・CTAアイコン |

### 6コマ漫画（7-B 初心者あるあるのみ）

```
┌───────┬───────┐
│  コマ1  │  コマ2  │
├───────┼───────┤
│  コマ3  │  コマ4  │
├───────┼───────┤
│  コマ5  │  コマ6  │
└───────┴───────┘
```

- 2列 × 3行の固定グリッド
- 各コマの境界線：2px・グレー（#BDBDBD）
- コマ番号は各コマ左上に小さく表示（22px・グレー）
- 吹き出しはコマ内に収める（はみ出し禁止）

### テキストブロック
- テキストブロックは左揃えを基本とする（中央揃えは見出しのみ）
- 箇条書きは「✅」「📌」「⚠️」など絵文字を先頭に使ってよい
- 1行に詰め込みすぎない（行間は1.4〜1.6を目安）

## 9. ファイル名

### 命名規則

```
[投稿種類コード]_[テーマ略称]_[枚数連番].png
```

### 投稿種類コード

| コード | 投稿種類 |
|---|---|
| `qa` | 7-A 問題形式 |
| `aa` | 7-B 初心者あるある |
| `ex` | 7-C 出来高・考え方解説 |
| `ap` | 7-D アプリ体験 |
| `rl` | 7-E リール |

### 例

| ファイル名 | 意味 |
|---|---|
| `qa_deadcross_01.png` | 問題形式・デッドクロス・1枚目 |
| `aa_sonkiri_01.png` | 初心者あるある・損切り・1枚目（全1枚） |
| `ex_idoheikin_02.png` | 解説・移動平均線・2枚目 |

### 禁止
- 日本語・スペース・記号（`- ( ) # & @`など）をファイル名に使わない
- `final` `new` `修正版` などの曖昧なサフィックスを付けない

---

## 10. 保存形式

| 項目 | 仕様 |
|---|---|
| 形式 | **PNG**（透過なし・RGB） |
| 色空間 | sRGB |
| 圧縮 | ロスレス（PNG標準） |
| 最大ファイルサイズ | 1枚あたり **8MB以内**（Instagram上限） |
| 推奨ファイルサイズ | 1枚あたり **3MB以内**（表示速度優先） |

### 注意
- JPEGは使わない（文字周辺に圧縮アーティファクトが発生するため）
- WebP・SVGはInstagramが非対応のため使わない
- アルファチャンネル（透過）は使わない（背景は必ず白で塗りつぶす）

## 11. 画像完成チェック

生成後、以下を全項目確認してから納品する。

### サイズ・形式
- [ ] 解像度が仕様通りか（フィード：1080×1350 / リール：1080×1920）
- [ ] PNG形式・sRGB・透過なしか
- [ ] ファイルサイズが8MB以内か
- [ ] ファイル名が命名規則に従っているか

### デザイン
- [ ] テーマカラー3色以内に収まっているか
- [ ] 1枚に使用フォントが3種類以内か
- [ ] 外周に余白（16〜24px以上）があるか
- [ ] 背景が白またはオフホワイトか

### テキスト
- [ ] 画像内テキストが1枚あたり3〜5点以内か（PROJECT_RULE.md § 11）
- [ ] キャプション・ハッシュタグが画像に含まれていないか（PROJECT_RULE.md § 11）
- [ ] 文字が小さすぎてスマホで読めないサイズになっていないか（本文28px以上）
- [ ] 断定的な投資表現が画像テキストに含まれていないか（PROJECT_RULE.md § 6）

### キャラクター・イラスト
- [ ] 手のポーズが判別できるか
- [ ] 吹き出しがコマまたは画面内に収まっているか
- [ ] リアル人物写真・特定銘柄・証券会社ロゴが含まれていないか

### 枚数・構成
- [ ] 投稿種類に対応した枚数になっているか（本ファイル § 2）
- [ ] 7-B 初心者あるあるが1枚・6コマ構成になっているか
- [ ] 各枚のゾーン（上部・中部・下部）が機能しているか

## 12. 納品ルール

画像生成が完了したら、
Codexは必ず以下の順番で納品する。

### 納品内容

① 作成した画像一覧

② 画像枚数

③ 各画像サイズ

④ ファイル名一覧

⑤ 完成チェック結果

### 完成チェック結果

- 画像枚数
- サイズ
- ファイル形式
- ファイル名
- PROJECT_RULE.mdとの整合性

を確認し、

問題がある場合は修正後に納品する。

問題がない場合のみ

「画像生成完了」

と報告する。
