# 0. Ver3.0での変更点（重要）

Ver2.1では「Canvaの有料編集機能を使わないため、ChatGPTが画像生成時点で文字を直接描画する」方式を採用していたが、
実運用で以下の問題が繰り返し発生し、約半月投稿が止まる原因になった。

・ChatGPTがタイトル・吹き出し・チャートラベルを空白のまま納品する
・文字を描いても誤字・崩れ・レイアウト逸脱が高頻度で発生する
・修正のたびに再生成が必要になり、ブランド一貫性が崩れる

このため、Ver3.0より以下の方式に変更する。

**ChatGPTは「文字を一切含まないイラスト」だけを生成する。**
**文字（タイトル・吹き出し・チャートラベル・CTA等）は、Claudeが`auto_layout.py`（Pythonの画像処理スクリプト）を使って、生成後のイラストに確定描画する。**

Canvaの有料編集機能を使わない、という制約は変わらないが、代替手段はCanvaではなくPythonによる自動テキスト合成である。
`auto_layout.py`は色解析でバッジ・白ボックス・チャート内カラーピル・チェックリストの丸アイコンなどを自動検出し、
アイコンに重ならない位置・統一されたフォントサイズで文字を合成する。イラストの形が多少ズレても追従する。

この変更により、image_request.md／canva_text.mdの位置づけを再度変更した（13-2／13-3参照、CONTENT_RULE.md側も要更新）。

# 1. このファイルの目的

このファイルは画像制作の仕様と品質基準を定義する。

**イラスト制作（文字なし）はChatGPTが担当する。**
**文字の合成（タイトル・吹き出し・チャートラベル・CTA）はClaudeが`auto_layout.py`で担当する。**

ChatGPTはGitHubを参照できないため、Claudeが本ファイルの内容をimage_request.mdへ転記し、自己完結した指示書として渡す。

# 3. 画像制作の手順

① STYLE_GUIDE.mdを読む（Claudeがimage_request.md作成時に参照）

② Brand_Reference/01_Bestを確認する

③ STYLE_GUIDE.md通りに、**文字を一切含まない**イラストを生成する

④ 生成後、ChatGPTは幅×高さの数値と、文字を入れる予定の空白箇所（バッジ／サブタイトル位置／吹き出し／チャート内ラベル位置／CTA）が意図通り空いているかを報告する

⑤ Claudeが`auto_layout.py`で、image_request.mdに記載された文言を各所に合成する

⑥ 本ファイルの納品前チェックリストで確認する

# 4. 出力サイズ（最重要）

フィード投稿：1080×1350px

リール：1080×1920px

トリミング禁止。

**1回の生成で複数カットをまとめた「コンタクトシート／グリッド画像」として書き出さない。**
**朝投稿3枚・リール3枚は、それぞれ独立した1枚のPNGファイルとして、1080×1350px（またはリールは1080×1920px）ちょうどで書き出す。**
**まとめて確認用に並べて表示された場合でも、納品時は1カットずつ個別ファイルでダウンロードする。**

**依頼する側（人間・Claude）も、1回のメッセージ・1回の生成依頼につき1カットのみを画像生成担当AIへ依頼すること。**
**複数カットの内容を1回のメッセージにまとめて送らない。** 画像生成モデルは、プロンプトで「1カットずつ」と指示しても、
複数カット分の情報が1つのメッセージに含まれていると、それらを1枚のコンタクトシートにまとめて出力してしまう挙動が
確認されている。これはプロンプトの書き方では制御しきれないモデル側の挙動であるため、**そもそも1メッセージに1カット分の
情報しか含めない**ことで防止する。image_request.mdは複数カットの一覧として作成してよいが、実際に画像生成担当AIへ
送る際は、カットごとに分割した個別のメッセージ（またはファイル）として、1カットずつ順番に依頼する。

出力後、幅×高さの数値を必ず報告すること。数値が指定と異なる場合は再出力する。

# 5. 参照画像の扱い（重要）

ChatGPTはGitHubを参照できないため、Brand_Reference/01_Bestの見本画像そのものを直接見ることができない。

そのため、人間が毎回Brand_Reference/01_Bestの見本画像を1〜2枚、ChatGPTへファイル添付で渡す運用とする。

添付画像が複数ある場合、必ず役割を区別して伝える。

・【絶対再現】Brand_Reference見本画像：レイアウト・キャラクター・クマ・背景・配色・吹き出し形状などを、この見本の通りに再現する対象

・上記以外（雰囲気・色味の参考用画像）：色味やタッチの参考にとどめ、レイアウトやキャラクターは変更しないこと

image_request.mdには

「添付された見本画像のうち、Brand_Reference見本画像を最優先で再現すること。それ以外の添付は雰囲気参考のみとし、レイアウト・キャラクターは変更しないこと」

と明記する。

# 6. 文字ゾーン設計ルール（Ver3.0で新設）

ChatGPTは文字を描かないが、**あとでClaudeが文字を正しく合成できるよう、文字を入れる場所を色で明確に区別して空けておく**必要がある。

・バッジ（濃い緑の角丸帯）：単色の濃い緑で塗り、その中に入れるアイコン（初心者マーク）は濃い緑のバッジ範囲内、左寄りに配置する。アイコンとバッジの右端の間に、タイトル文字を入れるための十分な余白（バッジ幅の60%以上）を確保する。

・サブタイトル：白箱を作らず、壁の背景色のまま余白として空ける（原則）。もし装飾上どうしても薄い色面を置きたい場合も、周囲の壁色に近い色にし、純白にはしない。

・チャート内ラベル（トレンド表示・出来高表示など）：色つきピル（角丸の色面）を単色で塗っておく。ピルの中に文字・記号・数字を入れない。ピルの色は、上昇＝緑系、下降＝赤系で統一する。

・出来高（棒グラフ）エリアには、赤・緑の棒グラフ以外の線（移動平均線の続き、グリッド線の誤描画など）を絶対に描かない。出来高エリアは棒グラフのみとする。

・吹き出し：**（Ver3.4で変更）吹き出し内の短いセリフは、image_request.mdで指定した文言をそのまま画像生成担当AIに描画させる。それ以外の吹き出し関連ルール（形・配置）は従来通り。**

・チェックリスト／凡例の丸アイコン（番号・チェックマーク・色ドット）：白い箱の中に等間隔で配置し、丸アイコンとテキスト用の余白がはっきり区別できるようにする（丸アイコン同士がテキスト行に重ならないよう、行の高さを均等にする）。

・下部のCTA・ナビゲーションバー：クリーム色のバーとして塗り、右側に矢印・キラキラ等の装飾アイコンを配置してよいが、装飾アイコンの左側に文字を入れるための余白を確保する。

# 6-3. 文字合成時の品質ルール（Ver3.4で新設・2026-07-16の失敗から）

auto_layout.pyで文字を合成する際、以下を必ず守る。

・**上部バッジ内の文言は「株初心者向け」で統一する。** カットごとのフック文言（タイトル）はバッジに入れず、
　バッジ直下のサブタイトル枠／見出し枠に入れる。バッジとタイトルは別の要素である。

・**バッジ直下に、白い縁取りの箱（サブタイトルボックス）を新たに作らない。** IMAGE_RULE.md 2章でも
　禁止しているが、実際の生成で繰り返し発生しているため再掲する。タイトル文言は壁の背景に直接乗せて
　描かせ、囲み枠を追加させない。image_request.mdで画像生成担当AIへ依頼する際、この点を明示的に注意書きすること。

・チェックリスト・番号リストの各項目テキストは、行の左端に寄せすぎない。行の余白全体を使って中央寄せに近い
　バランスで配置する。

・**バッジ直下の枠（サブタイトル・チェックリスト等）は、枠の大きさに対してできるだけ文字を大きく、
　枠を使い切るサイズで配置する。** 小さい文字を隅に置いただけの状態で納品しない。

・**下部のCTA・ナビゲーションバーの文言は、矢印アイコンを避けつつ中央寄せで配置する。** 左詰めにしない。

・**吹き出しは、image_request.mdで指定したセリフが画像生成担当AI側で正しく描画されているか、
　画像内に存在するものを一つ残らず確認する。** 特にリール（問題／答え／解説）は吹き出しが複数枚にまたがる
　ことがあるため、納品前に画像内の吹き出しの数を目視で数え、空白・誤字・崩れがないか確認する。
　万が一空白・崩れがあれば、その吹き出しだけClaude側でauto_layout.pyにより文字を合成して補う。

・**6コマ漫画のセリフは、他のカットより一段階デザイン性を上げる。** 単色の小さい文字で済ませず、
　太字・適度な強調色（オレンジ等）を使い、コマごとの感情の起伏が伝わるサイズ感にする。

# 6-2. 線の交差を含むチャートの指示方法（Ver3.1で新設）

ゴールデンクロス・デッドクロスなど、2本の移動平均線が交差する形を描かせる際、
「ゴールデンクロスを描いて」のようにパターン名だけを伝えると、線が実際には交差していないチャートが生成されることが多い。

**線が交差するカットをimage_request.mdに書く際は、必ず交差の前・交差点・交差後の3点で、
それぞれの線の上下関係を明示的に文章化すること。**

例（ゴールデンクロスの場合）：

・チャート左側：短期線（25日線）が長期線（75日線）より下にある
・交差点：2本の線が触れる／重なる
・チャート右側：短期線（25日線）が長期線（75日線)より上にある

パターン名（ゴールデンクロス／デッドクロス等）は説明として併記してよいが、
**線の位置関係の明示的な文章を必ず併記する。名称だけに頼らない。**

**さらに、交差後に一方の線が消える／もう一方の線の色に変わってしまう不具合が確認されている
（特に、下に潜る方向へ交差する場合＝デッドクロスで発生しやすい）。**
**これを防ぐため、以下も必ず併記すること。**

例（デッドクロスの場合）：

・25日線（オレンジ）は交差点で終わらない。交差後も一本の連続した線として、チャート右端まで必ず描く。
・交差後は75日線（緑）の下側を、そのまま右端まで連続して伸ばす。
・25日線は途中で消してはいけない。25日線が途中で75日線の色（緑）に変化してはいけない。
・2本の線は、チャートの左端から右端まで、それぞれ同じ色で必ず1本ずつ存在し続ける。

可能であれば、文章に加えて簡易的な模式図（例：🟠──╲　　╱──🟢／🟢──╱────╲──🟠）も
image_request.mdに併記すると、さらに伝わりやすくなる。

**さらに、上記の文章による指示だけでは改善しなかったため（2026-07-15検証）、
`charts/chart_generator.py`で「正解チャート」の参考画像を先に生成し、
ブランド見本画像と一緒にChatGPTへ添付する方式を正式な標準手順とする。**

手順：

1. Claudeが`charts/chart_generator.py`で、そのカットのパターン（dead_cross／golden_cross）に合わせた参考チャート画像を生成する
2. 添付1（ブランド見本画像）・添付2（生成した参考チャート画像）の2枚をChatGPTへ添付する
3. image_request.mdで「チャート部分は添付2を完全再現。それ以外は添付1を再現」と役割を明確に分けて指示する

`chart_generator.py`は、ろうそく足・移動平均線・出来高に加えて、交差点を示す
「ふんわり光る丸＋キラキラ」の強調演出（出来高側にも同じ演出）を自動で描画する。
この演出は初心者向けの視認性向上のために採用しており、ChatGPTにもそのまま模写させてよい。

# 8. 修正時のルール

修正とは新しい画像を作ることではない。

ブランドを維持したまま必要箇所だけ修正することである。

□ 修正対象以外を変更していない

□ キャラクターを変更していない

□ クマを変更していない

□ レイアウトを変更していない

□ 背景を変更していない

□ 色を変更していない

□ 世界観を変更していない

チャート修正時は特に注意する。

ローソク足のサイズ・本数・太さは変更禁止。

変更してよいのは色・背景・ラベル用ピルの位置のみ。

文字はChatGPT側では一切修正しない（Claude側のauto_layout.pyで再合成する）。

# 11. 最重要ルール

ChatGPTは画像生成AIではない。

**文字を含まないイラストの生成担当である。**

迷った場合は新しいデザインではなくブランド維持を優先する。

説明より図を信用しない。

図・チャート・矢印・ラベル用ピル・吹き出し・文字ゾーンの6つがすべて「文字なしで」正しく配置されて初めて完成とする。

**文字はChatGPTが描くとほぼ必ず誤字・欠落・崩れが起きるため、ChatGPT側では絶対に描画しない。**

シリーズ画像と並べたときに違和感がある場合は納品してはいけない。
